# Lab 7 - Python FastAPI Project (well-done level)

API на FastAPI с тремя эндпоинтами: рекурсивный обход дерева счётов, профилировщик памяти через `tracemalloc` и вычисление числа π методом Лейбница. В комплекте - готовый HTML-фронтенд.

---

## Структура проекта

```
lab7_welldone_level.py/
├── package/
│   ├── __init__.py                    # реэкспорт из lab4, lab5, lab6
│   ├── lab4.py                        # get_matches_result_recursion
│   ├── lab5.py                        # memory_decorator
│   └── lab6.py                        # pi_leibniz (генератор)
├── example_functions_for_decorator.py # 5 функций с @memory_decorator
├── main.py                            # FastAPI приложение
└── frontend.html                      # готовый веб-интерфейс
```

---

## Установка и запуск

**Требования:** Python 3.10+

```bash
# Установить зависимости
pip install fastapi uvicorn

# Запустить сервер
python main.py
# или напрямую через uvicorn
uvicorn main:app --reload
```

Сервер поднимается на `http://127.0.0.1:8000`.

Документация Swagger доступна по адресу `http://127.0.0.1:8000/docs`.

Открыть `frontend.html` в браузере - он обращается к серверу напрямую.

## API

### GET `/get_matches_result/{k}`

Генерирует все возможные пути развития счёта матча от `0:0` до момента, когда одна из команд набирает ровно **k** очков. Алгоритм - рекурсивный обход бинарного дерева состояний.

**Параметры пути:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `k` | `int` | Победный счёт (≥ 1) |

**Пример запроса:**
```
GET /get_matches_result/2
```

**Пример ответа:**
```json
{
  "result": [
    ["0:0", "1:0", "2:0"],
    ["0:0", "1:0", "1:1", "2:1"],
    ["0:0", "0:1", "1:1", "2:1"],
    ["0:0", "0:1", "0:2"],
    ["0:0", "1:0", "1:1", "1:2"],
    ["0:0", "0:1", "1:1", "1:2"]
  ]
}
```

`result` - массив путей; каждый путь - упорядоченный список состояний счёта.

### GET `/use_memory_decorator/{index_func}`

Вызывает одну из 5 функций, обёрнутых в декоратор `@memory_decorator`. Декоратор запускает `tracemalloc` вокруг вызова и возвращает результат функции вместе со статистикой потребления памяти.

**Параметры пути:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `index_func` | `int` | Номер функции от 1 до 5 |

**Query-параметры** (зависят от выбранной функции):

| Параметр | Тип | Для функций |
|----------|-----|-------------|
| `count` | `int` | 1, 2, 3, 4 |
| `text` | `str` | 5 |

**Доступные функции:**

| № | Функция | Аргумент | Описание |
|---|---------|----------|----------|
| 1 | `list_of_squares(n)` | `count` | Список квадратов `[0, 1, 4, …, (n-1)²]`. Память растёт линейно. |
| 2 | `generator_of_squares(n)` | `count` | То же через генераторное выражение, приведённое к списку. |
| 3 | `fibonacci_recursive(n)` | `count` | Числа Фибоначчи через рекурсию без мемоизации. Пик памяти резко растёт из-за стека вызовов. **Рекомендуется n ≤ 25.** |
| 4 | `matrix_multiply(size)` | `count` | Умножение двух матриц size×size. Память растёт как O(n²). |
| 5 | `word_frequency(text)` | `text` | Частота слов в строке. Возвращает словарь `{слово: кол-во}`. |

**Пример запроса:**
```
GET /use_memory_decorator/1?count=500
GET /use_memory_decorator/5?text=hello+world+hello+python
```

**Пример ответа:**
```json
{
  "result": [0, 1, 4, 9, 16, 25],
  "memory_current": 4280,
  "memory_peak": 8560,
  "describe": "Память: текущая 4280 байт | пик 8560 байт"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `result` | any | Возвращаемое значение функции |
| `memory_current` | `int` | Потребление памяти в момент окончания функции (байт) |
| `memory_peak` | `int` | Пиковое потребление памяти за время работы (байт) |
| `describe` | `str` | Человекочитаемая строка с обоими значениями |

### GET `/get_pi_numbers/{iterations}`

Вычисляет число π по ряду Лейбница:

```
π/4 = 1 - 1/3 + 1/5 - 1/7 + ...
```

Функция `pi_leibniz` реализована как генератор, отдающий промежуточные приближения. Сервер итерирует генератор и возвращает финальное значение.

**Параметры пути:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `iterations` | `int` | Количество итераций ряда (≥ 1) |

**Пример запроса:**
```
GET /get_pi_numbers/100000
```

**Пример ответа:**
```json
{
  "result": 3.1415826535897198
}
```

Точность: при 100 000 итерациях погрешность составляет порядка `10⁻⁵`. Ряд Лейбница сходится медленно - для 10 верных знаков потребуется порядка 10⁹ итераций.

## Модули пакета

### `lab4.py` - `get_matches_result_recursion(k)`

Рекурсивная функция-helper строит все пути в дереве состояний счёта. На каждом шаге счёт увеличивается на 1 у одной из двух команд. Рекурсия останавливается, когда `max(a, b) == k`.

### `lab5.py` - `memory_decorator`

Декоратор использует `functools.wraps` для сохранения метаданных функции и флаг `_is_running` для предотвращения рекурсивного запуска трассировки (необходимо для `fibonacci_recursive`, который вызывает себя повторно). `tracemalloc.start()` / `tracemalloc.stop()` оборачивают вызов функции через `try/finally`.

### `lab6.py` - `pi_leibniz(iterations)`

Генератор: на каждой итерации прибавляет следующий член ряда `(-1)^k / (2k+1)` и отдаёт текущее приближение `pi * 4`. Позволяет наблюдать сходимость ряда по шагам.

## Фронтенд

Файл `frontend.html` - одностраничный интерфейс без зависимостей (только Google Fonts). Открывается прямо из файловой системы.

Поле **BASE URL** в верхней части страницы позволяет переключить адрес сервера.

Три секции соответствуют трём эндпоинтам. Для `/use_memory_decorator` предоставлен интерактивный список функций с описанием - при выборе автоматически появляется нужное поле ввода (`count` или `text`). Результаты отображаются в виде карточек: для памяти - текущая и пиковая в байтах с конвертацией в КБ/МБ, для π - вычисленное значение, погрешность и процент точности, для матчей - количество путей и все пути в виде тегов.
