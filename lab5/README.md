## 1. Условие задач

### Задача 1. Замыкание для чтения n байт

Реализовать замыкание, которое получает n байт (или символов) из заданного источника данных. Каждый вызов внутренней функции должен возвращать следующий фрагмент длиной n, сохраняя позицию между вызовами.

**Требования:**
- Внешняя функция принимает источник данных и размер фрагмента n
- Внутренняя функция не принимает аргументов и возвращает очередной фрагмент
- Текущая позиция хранится в замыкании через `nonlocal`

### Задача 2. Декоратор для отслеживания памяти

Реализовать декоратор, который измеряет использование памяти декорируемой функцией с помощью `sys.getsizeof()`. Применить декоратор к замыканию из задачи 1.

**Требования:**
- Декоратор выводит сообщение до вызова функции
- После вызова выводит объём памяти возвращаемого значения в байтах
- Применяется к замыканию с помощью `@`

## 2. Описание проделанной работы

### 2.1. Замыкание `make_read`

Функция `make_read` создаёт замыкание для последовательного чтения фрагментов из строки. Переменная `offset` хранит текущую позицию и захватывается внутренней функцией `read` через `nonlocal`.

```python
def make_read(source, n):
    offset = 0

    def read():
        nonlocal offset
        chunk = source[offset: offset + n]
        offset += n
        return chunk

    return read
```

Механизм работы:
- `make_read(source, n)` - внешняя функция, создаёт «ячейки памяти»: `offset`, `source`, `n`
- `read()` - внутренняя функция-замыкание, захватывает все три переменные
- `nonlocal offset` - разрешает изменять захваченную переменную
- `return read` - возвращает функцию, не вызывая её

### 2.2. Декоратор `memory_decorator`

Декоратор оборачивает функцию в `wrapper`, который выводит сообщение до вызова и измеряет память результата с помощью `sys.getsizeof()` после вызова.

```python
import sys

def memory_decorator(func):
    def wrapper():
        print("Начало выполнения функции")
        result = func()
        print(f"Память, использованная функцией: {sys.getsizeof(result)} байт")
        return result
    return wrapper
```

`memory_decorator` сам является замыканием - `wrapper` захватывает `func` из внешней функции.

### 2.3. Применение декоратора к замыканию

Декоратор `@memory_decorator` применяется к внутренней функции `read` прямо внутри тела `make_read_and_decorator`:

```python
def make_read_and_decorator(source, n):
    offset = 0

    @memory_decorator
    def read():
        nonlocal offset
        chunk = source[offset: offset + n]
        offset += n
        return chunk

    return read
```

Это демонстрирует, что декоратор - частный случай замыкания: при каждом вызове `read()` декоратор перехватывает выполнение, добавляя поведение до и после.

## 3. Результаты выполнения

### 3.1. Задача 1 - чтение без декоратора

```python
read5 = make_read("hello world", 5)

for i in range(4):
    print(read5())
```

**Вывод:**
```
hello
 worl
d

```

Первые два вызова вернули фрагменты по 5 символов. Третий - остаток `"d"` (1 символ). Четвёртый - пустую строку, так как данные закончились.

### 3.2. Задача 2 - декоратор на обычной функции

```python
@memory_decorator
def create():
    return [1] * 100

create()
```

**Вывод:**
```
Начало выполнения функции
Память, использованная функцией: 856 байт
```

Список из 100 целых чисел занимает 856 байт. `sys.getsizeof()` возвращает размер самого объекта списка без глубокого подсчёта элементов.

### 3.3. Декоратор применён к замыканию

```python
read8 = make_read_and_decorator(
    "qwertyuiopadfghjkldfhjghidfgdhgidfgidhgdhighdg", 13
)

for i in range(5):
    print(read8())
```

**Вывод:**
```

qwertyuiopadf
Начало выполнения функции

ghjkldfhjghid
Начало выполнения функции
Память, использованная функцией: 62 байт
fgdhgidfgidhg

Память, использованная функцией: 56 байт

Начало выполнения функции

```

## 4. Выводы

- Замыкание `make_read` сохраняет состояние (`offset`) между вызовами без использования классов или глобальных переменных
- Декоратор `memory_decorator` является замыканием, захватывающим `func` и добавляющим поведение до и после её вызова
- Декоратор `@memory_decorator` успешно применён к внутренней функции `read` - это подтверждает, что замыкания и декораторы используют один механизм захвата переменных
- `sys.getsizeof()` измеряет размер объекта
